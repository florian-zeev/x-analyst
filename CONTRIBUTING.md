# Contributing

Thanks for your interest in improving X Analyst.

Before opening a pull request:

1. Keep changes focused and small.
2. Follow the existing code style and project patterns.
3. Do not add production dependencies without discussing them first.
4. Do not include secrets, private data, or customer-specific configuration.
5. Run the narrowest meaningful validation command before submitting.

By submitting a contribution, you agree that your contribution is licensed under
the repository license and may also be used by the copyright holder under
separate commercial terms.
